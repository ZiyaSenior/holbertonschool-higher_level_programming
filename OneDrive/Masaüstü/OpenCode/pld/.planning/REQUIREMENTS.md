# Requirements: Student Dashboard UI Refresh

**Defined:** 2026-03-10
**Core Value:** Students should land on a calm, clear dashboard that immediately helps them see what matters next and act on it without distraction.

## v1 Requirements

### Dashboard Flow

- [ ] **DASH-01**: Student can see one clear top-of-page next action when the dashboard loads
- [ ] **DASH-02**: Student can scan the dashboard without seeing a stats-first or admin-style summary strip above primary actions
- [ ] **DASH-03**: Student sees calm empty states that point to a useful next step instead of a dead end

### Sessions

- [ ] **SESS-01**: Student can see their most relevant session first with clear topic, date, and start time
- [ ] **SESS-02**: Student can distinguish between join-now, upcoming, and other joinable session states on the dashboard
- [ ] **SESS-03**: Student can join an eligible future session directly from the dashboard
- [ ] **SESS-04**: Student sees consistent and understandable session time formatting across dashboard sections

### Practice

- [ ] **PRAC-01**: Student can start AI practice from a persistent dashboard action near the top of the page

### Announcements

- [ ] **ANNC-01**: Student can read a compact list of the latest announcements with short previews
- [ ] **ANNC-02**: Student can navigate from the dashboard to the full announcements page

### Preferences

- [ ] **PREF-01**: Student can see their current PLD preference in a compact summary instead of a dominant full-size panel
- [ ] **PREF-02**: Student can edit their PLD preference from the dashboard without triggering the current error path
- [ ] **PREF-03**: Student can see available PLD start times clearly when selecting or reviewing a preference
- [ ] **PREF-04**: Student receives clear success or failure feedback after saving a PLD preference and sees the updated state reflected in the dashboard

### Responsiveness

- [ ] **RESP-01**: Student can complete the primary dashboard actions on desktop and mobile layouts without hidden CTAs or broken reading order

## v2 Requirements

### Guidance

- **GUID-01**: Student sees a personalized next-step message based on recent activity and session state
- **GUID-02**: Student sees a gentle progress cue that encourages momentum without turning the dashboard into a KPI board
- **GUID-03**: Student sees a lightweight preparation hint attached to the next session

## Out of Scope

| Feature | Reason |
|---------|--------|
| Mentor or admin dashboard redesign | This project is only for the student dashboard route |
| Full analytics or achievement-heavy dashboard surfaces | The goal is clarity and next-action guidance, not more metrics |
| Broad backend or API redesign | Existing contracts should remain in place unless a small fix is needed for the PLD preference bug |
| Product-wide navigation overhaul | This effort should stay scoped to the student dashboard experience |
